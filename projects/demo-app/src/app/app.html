Installation

Install from npm:

Terminal 
npm install ng-track-event-directive

Peer dependencies: @angular/core and @angular/common v21.2+

Developer Onboarding

  1. Install the package and confirm your Angular version is compatible.
  2. Create a single analytics adapter and register it with provideTrackingAdapter().
  3. Import TrackEventDirective in the standalone components that need tracking.
  4. Use event suffixes consistently: :clicked, :hovered, :viewed.
  5. Verify in browser console first, then in your provider dashboard (Mixpanel Live View).
  6. Document your event naming convention in your app repo so product and engineering align.

Note: npm package documentation is shown from projects/tracking/README.md, so keep that file comprehensive and synced with this site.

Quick Start

1. Provide an adapter

app.config.ts 
import { ApplicationConfig } from '@angular/core';
import { provideTrackingAdapter, TrackingAdapter } from 'ng-track-event-directive';

const adapter: TrackingAdapter = {
  track(eventName: string, data?: unknown): void {
    console.log('track', eventName, data);
  },
};

export const appConfig: ApplicationConfig = {
  providers: [provideTrackingAdapter(adapter)],
};

2. Import directive in component

page.component.ts 
import { Component } from '@angular/core';
import { TrackEventDirective, trackConfig } from 'ng-track-event-directive';

@Component({
  selector: 'app-page',
  imports: [TrackEventDirective],
  template: '<button [trackEvent]="saveTrack">Save</button>',
})
export class PageComponent {
  saveTrack = trackConfig('page-save:clicked', { source: 'toolbar' });
}

3. Realtime Mixpanel setup

mixpanel-adapter.ts 
import mixpanel from 'mixpanel-browser';
import { provideTrackingAdapter, TrackingAdapter } from 'ng-track-event-directive';

mixpanel.init('YOUR_MIXPANEL_PROJECT_TOKEN', { track_pageview: true });

const mixpanelAdapter: TrackingAdapter = {
  track(eventName, data) {
    mixpanel.track(eventName, (data ?? {}) as Record<string, unknown>);
  },
};

Integration Examples and Prerequisites

This directive is provider-agnostic. You can connect any analytics SDK by implementing TrackingAdapter. Mixpanel is fully documented below, with more providers listed as upcoming.

Prerequisites

  • Angular 21.2+ with standalone components.
  • Install this package: npm install ng-track-event-directive
  • Install provider SDK in your app (example: mixpanel-browser).
Mixpanel SDK install 
npm install mixpanel-browser

Mixpanel (Complete example)

mixpanel-adapter.ts 
import mixpanel from 'mixpanel-browser';
import { TrackingAdapter } from 'ng-track-event-directive';

mixpanel.init('YOUR_MIXPANEL_PROJECT_TOKEN', {
  track_pageview: true,
  persistence: 'localStorage',
});

export const mixpanelAdapter: TrackingAdapter = {
  track(eventName: string, data?: unknown): void {
    mixpanel.track(eventName, (data ?? {}) as Record<string, unknown>);
  },
};

Trigger Suffix Conventions

Event suffix Trigger Default once
:clicked click false
:hovered hover false
:viewed view (IntersectionObserver) true

API Reference

  • TrackEventDirective: Core directive, selector [trackEvent].
  • trackConfig(): Type-safe factory for TrackConfig objects.
  • provideTrackingAdapter(): Registers your analytics adapter.
  • TRACKING_ADAPTER: DI token for adapter injection.
  • parseTriggerFromEvent(): Parses suffix to track trigger.
  • TrackTrigger: 'click' | 'view' | 'hover' | 'unknown'.

Verification + Troubleshooting

  1. Open DevTools Console and look for logs prefixed with [ng-track-event demo].
  2. If using Mixpanel, confirm requests to api-js.mixpanel.com are not blocked.
  3. Double-check event names include a supported suffix.
  4. If no adapter is provided, directive intentionally no-ops without throwing errors.
